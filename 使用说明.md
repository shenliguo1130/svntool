# SVN 工具箱 · 使用说明

一个 macOS 原生 SVN GUI 客户端，基于 Tkinter + PyInstaller 打包，体积小、操作直观。

---

## 一、系统要求

- macOS 11+ (Big Sur 及以上)
- 需提前通过命令行检出 SVN 仓库（app 不提供 checkout 功能）

---

## 二、环境准备

### 1. 安装 Homebrew（如未安装）

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

### 2. 安装 svn 命令行工具

```bash
brew install svn
```

> 验证安装：`svn --version` 能正常输出版本信息即可。

### 3. 检出 SVN 仓库

```bash
svn checkout <repository_url> /path/to/local/repo
```

> ⚠️ 本 app 仅识别已通过命令行检出的 SVN 工作副本，自身不提供 checkout 功能。
> 建议将仓库检出到 app 所在目录的子目录中（默认扫描当前目录及子目录，深度 3 层）。

---

## 三、安装 app

将 `SVN工具箱.app` 拖入 `/Applications` 文件夹，双击运行。

> 首次启动若弹出"无法打开，因为无法验证开发者"，右键 → 打开 → 打开。

---

## 四、界面说明

```
┌─────────────────────────────────────────────┐
│ 仓库列表              输  出  面  板          │
│ 📁 repo-1            [仓库切换到:repo-1]...  │
│ 📁 repo-2            ?     未版本控制 file1  │
│ 📁 repo-3            M     已修改   file2    │
│                                              │
│  [刷新仓库列表]   [刷新状态][解决冲突]...     │
│                          [撤销][更新][提交]   │
└─────────────────────────────────────────────┘
```

### 左侧：仓库列表

- 自动扫描当前目录及子目录（3 层深度）下的 `.svn` 工作副本
- 启动时异步扫描，不阻塞 UI

### 右侧：输出面板

- 实时显示操作日志，**不换行**，长路径需手动拉宽窗口查看
- 切换仓库时自动清屏并刷新 `svn st`

### 底部按钮

- **刷新仓库列表**（左）— 重新扫描 `.svn` 目录
- **刷新状态**（右）— 重新执行 `svn st`
- **解决冲突**（右）— 解决冲突文件
- **撤销**（右）— 撤销本地修改
- **更新**（右）— `svn up`
- **提交**（右）— `svn ci`

## 五、操作详解

### 1. 刷新状态

点击后立即输出 `开始获取仓库状态...`，随后显示 `svn st` 结果，状态码转中文。

### 2. 更新

点击后立即输出 `开始拉取更新...`，随后流式显示 `svn up` 过程，完成后自动刷新状态。

### 3. 提交

弹框内：
- 顶部：全选复选框 + 分类筛选（`?` 未版本控制 / `M` 已修改 / `!` 缺失 / `A` 已添加 / `D` 已删除 / `~` 类型变更 / `L` 锁定）
- 中部：每个文件前的复选框（可单选）
- 底部：输入提交备注（留空使用时间戳）→ 提交

逻辑：
- 选中的 `?` 文件自动 `svn add`
- 选中的 `!` 文件自动 `svn rm`
- 选中的 `M` 等文件直接 `svn ci -m "备注" <files>`
- **C 冲突文件不出现在提交列表**，需先解决冲突
- **X 外部引用文件不出现**，不可提交

### 4. 撤销

弹框内列出所有 `M` 文件，复选选择要撤销的，点击「撤销选中」。

> ⚠️ 撤销后无法恢复！

### 5. 解决冲突

弹框内列出所有 `C` 文件：
- **使用他人版本** — 丢弃本地，用仓库版本覆盖
- **使用自己版本** — 保留本地，丢弃仓库变更

完成后文件状态由 `C` 变为 `M`，可在提交弹框中提交。

## 六、快捷键

| 按键 | 功能 |
|------|------|
| `↑` `↓` | 切换仓库 |
| `Enter` | 执行当前聚焦的按钮 |
| `Esc` | 取消按钮聚焦 |

> 弹框打开时，快捷键不响应弹框外的操作。

## 七、状态码对照

| 符号 | 中文 | 处理方式 |
|------|------|---------|
| `?` | 未版本控制 | svn add |
| `M` | 已修改 | 直接提交 |
| `!` | 缺失 | svn rm |
| `A` | 已添加 | 直接提交 |
| `D` | 已删除 | 直接提交 |
| `C` | 冲突 | 需解决冲突 |
| `~` | 类型变更 | 直接提交 |
| `L` | 锁定 | 直接提交 |
| `X` | 外部引用 | 不可直接提交 |

## 八、常见问题

### Q1: 启动后仓库列表为空？

- 检查 SVN 工作副本是否有 `.svn` 目录
- 默认扫描深度为 3 层，超出范围的仓库不会被识别
- 启动 app 的位置需要在工作副本的上级目录

### Q2: 启动时提示"未检测到 svn 命令行工具"？

按本文档"二、环境准备"安装 svn 后重启 app。

### Q3: 提交失败？

- 有冲突文件 → 先解决冲突
- 备注为空 → 留空会自动使用时间戳作为备注
- 服务器要求强制备注 → 手动填写具体内容

### Q4: 启动很慢？

PyInstaller `--onefile` 打包的固有特性，每次启动需要解压。如有更高要求需在沙箱外用 `--onedir` 模式重新打包。

### Q5: 日志框看不到完整文件路径？

默认不换行，鼠标拖动窗口边缘拉宽即可，或直接看日志下方的滚动条（隐藏但可用鼠标滚轮或触摸板）。

---

## 九、反馈

如遇问题，请检查：
1. `svn --version` 是否正常
2. 仓库是否已正确检出
3. 启动 app 的位置是否在工作副本上级目录
